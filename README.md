# Aiti Guru – Тестовое задание (React / TypeScript)

### Описание проекта

Тестовое задание для позиции фронтенд‑разработчика (React) в Aiti Guru.  
Приложение – административная панель для работы с товарами: авторизация, список товаров, сортировка, поиск и добавление новых позиций.

---

### Функционал

- **Экран авторизации**
  - Форма входа: логин, пароль, чекбокс «Запомнить меня».
  - Валидация обязательных полей и отображение ошибок под инпутами.
  - Авторизация через DummyJSON Auth.
  - Запоминание сессии:
    - при включённом чекбоксе – токен хранится в `localStorage` (живёт после закрытия браузера);
    - при выключенном – в `sessionStorage` (сбрасывается при закрытии вкладки).

- **Страница с товарами**
  - Загрузка списка товаров из DummyJSON Products API.
  - Таблица, визуально соответствующая макету в Figma (колонки: id, наименование, цена, рейтинг, бренд/вендор, артикул и т.д.).
  - Индикатор загрузки / прогресс‑бар на время запросов.
  - Сортировка по ключевым столбцам (например, цена, рейтинг, название) с сохранением состояния сортировки.
  - Поиск товаров через API (серверный поиск, а не только фильтрация на клиенте).
  - Добавление товара:
    - кнопка «Добавить» открывает форму/модальное окно;
    - поля: наименование, цена, вендор (бренд), артикул;
    - после сохранения товар добавляется в локальное состояние (без реального POST в API);
    - отображается toast‑уведомление об успешном добавлении.
  - Логика отображения рейтинга:
    - рейтинг ниже 3 подсвечивается красным цветом.

---

### Технологический стек

- **React** 18+ (функциональные компоненты и хуки)
- **TypeScript** (строгая типизация, типы для API и сущностей товаров)
- **Vite** (сборка и dev‑сервер)
- **Zustand** (стейт‑менеджер для auth и товаров)
- **Tailwind CSS** (стилизация интерфейса)
- **ESLint** (flat‑конфиг для TS/React)
- **Prettier** (форматирование кода)
- **Husky + lint-staged** (pre‑commit проверка кода)

Приложение ориентировано на **актуальную версию Google Chrome**.

---

### Состояние и архитектура

- Отдельные стора Zustand:
  - `auth` – токен, пользователь, флаг «запомнить меня»;
  - `products` – список товаров, состояние загрузки, ошибка, сортировка, параметры поиска.
- Строгие типы для:
  - сущности товара;
  - ответов DummyJSON Auth и Products.

---

### Качество кода

- **ESLint**:
  - flat‑конфиг для TypeScript и React;
  - интеграция с Prettier (`eslint-plugin-prettier/recommended`).
- **Prettier**:
  - единые правила форматирования (`.prettierrc`).
- **Husky + lint-staged**:
  - pre‑commit хук запускает `lint-staged`;
  - для изменённых файлов:
    - TS/JS/TSX/JSX – `eslint --fix` и `prettier --write`;
    - CSS/MD/JSON – `prettier --write`.

### Как запустить проект локально

```bash
git clone <repo-url>
cd aiti-guru-test
npm install
npm run dev
```

После запуска dev‑сервера Vite открой в браузере (Google Chrome) адрес, указанный в консоли, и проверь:

- экран авторизации;
- страницу товаров после успешного входа;
- сортировку, поиск и добавление товара с toast‑уведомлением.
